# 一路凯歌官网本地发布工作台

这个工作台用于半自动发布 `www.yilukaige.com` 新闻文章。

原则：

- GitHub 仓库仍然是唯一内容源。
- 工作台只在本机运行，不放到公网。
- 先生成本地页面和检查，再手动确认部署。
- 部署后会同步服务器、检查线上页面、提交 IndexNow，并尝试百度普通收录。

## 启动方式

在项目目录运行：

```powershell
npm run workbench
```

浏览器打开：

```text
http://127.0.0.1:4388/
```

## 使用流程

1. 填写文章标题、URL slug、分类、摘要、关键词和 SEO 描述。
2. 添加正文小节，每个小节可以填写段落、要点和图片。
3. 图片填写本机完整路径，工作台会复制到 `assets/`。
4. 添加 FAQ，至少 3 条，建议 4 条。
5. 点击“检查草稿”。
6. 点击“生成本地页面”。
7. 打开本地预览确认页面、图片、FAQ 正常。
8. 点击“提交并部署”。

## 自动完成的事情

生成本地页面时会自动：

- 把文章追加到当前最新的 `tools/news-articles-*.mjs`
- 复制图片到 `assets/`
- 生成文章详情页
- 更新 `news.html`
- 更新新闻分页
- 更新首页最新文章
- 更新 `sitemap.xml`
- 更新 `feed.xml`
- 更新 `llms.txt`
- 检查本地链接和结构化数据

部署时会自动：

- 提交 GitHub
- 推送到 `main`
- 同步静态文件到 `/opt/yilu-kaige`
- 检查线上首页、新闻页、文章页、sitemap、feed、llms
- 提交 IndexNow
- 尝试百度普通收录

## 注意

如果工作台提示“当前本地有未提交改动”，先不要继续生成文章。这个保护是为了避免把无关内容一起发布。

如果百度返回 `over quota`，说明当天普通收录额度用完，记录即可，第二天再提交。
