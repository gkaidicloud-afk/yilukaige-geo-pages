2026-05-31GA4 分析

GA4 流量来源有用户、会话、事件三层，AI 搜索归因别只盯一个 source/medium

AI 搜索带来的客户路径经常不是“点一下就成交”，而是先被问答种草，再多次回访，再在别的会话里转化。如果你的报表只看一个 source/medium，很容易把这条路看扁。

GA4 不是只有一套来源口径

Google Analytics 官方帮助文档把流量来源分成 user-scoped、session-scoped 和 event-scoped 三层。简单说，`First user source / medium` 代表一个用户第一次怎么来的；`Session source / medium` 代表这次会话怎么开始；事件层来源则用于关键事件归因，会受你选定的 attribution model 影响。

对 AI 搜索来说，这种区分尤其重要。因为用户第一次可能是从 ChatGPT、Google AI Mode 或 Kimi 搜索结果里认识你，但真正提交表单的时候，可能已经是第二次、第三次访问，甚至是直接输入网址回来。

如果只看一个报表，会错判什么

只看 `Session source / medium`，你会以为很多最终成交都是 direct 或 brand search，低估前面 AI 搜索的启发作用；只看 `First user source / medium`，你又可能高估 AI 的成交贡献，看不到中间是不是还经历了公众号、邮件或销售跟进；只看关键事件来源，则可能忽略了品牌最初是在哪个 AI 入口被看到。

一路凯歌做 GEO 流量复盘时，更建议把这三层放在同一套月报里：第一层看 AI 是否在带来新用户，第二层看 AI 进入的网站访问质量，第三层看关键转化最终被哪些渠道拿到归因。

怎么把它落到站点实践里

如果你正在做资讯更新和 FAQ 补全，可以同步规范所有站外分发链接的 UTM 规则，并把电话点击、表单提交、微信复制、预约按钮点击等定义成关键事件。这样当 AI 搜索把用户带到不同文章页时，你就不只是看到流量，还能看到这些流量后续有没有继续阅读和转化。

这也是为什么 Search Console 的 AI Mode 数据 只能算第一段。看见了不等于成交，成交了也不一定是第一次那一下完成的。GEO 想进入可复盘阶段，GA4 这三层来源维度必须拆清楚。

要点总结

  • GA4 的来源维度至少有用户、会话、事件三层，不是一张来源报表能解释完的。
  • AI 搜索常见路径是先种草、再回访、后转化，三层口径正好对应这条链路。
  • GEO 月报应同时看 First user、Session 和关键事件来源维度。

常见问题

为什么 AI 搜索流量不能只看 Session source / medium？
因为首次获客、当前会话来源和关键事件归因可能是三套不同口径。只看其中一个，很容易忽略用户第一次被 AI 种草、后来通过其他入口回来成交的路径。
做 GEO 月报最少要看哪三项？
建议至少同时看 First user source / medium、Session source / medium，以及关键事件或转化相关的 event-scoped 来源维度。
